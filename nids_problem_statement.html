01

Problem Statement

Modern computer networks carry billions of packets daily. Embedded within this normal traffic are malicious packets — crafted by attackers to overwhelm servers, steal credentials, escalate privileges, or establish unauthorized remote access. The challenge is identifying these attacks automatically, accurately, and in real time.

Traditional signature-based intrusion detection systems maintain lists of known attack patterns. They are effective against known threats but fail against zero-day exploits and novel attack variants. They also require constant manual updates and produce high false-positive rates on modern high-throughput networks.

This project addresses the problem by framing network intrusion detection as a supervised binary classification task: given a connection record described by 41 numerical and categorical features (protocol, duration, byte counts, error rates, etc.), predict whether the connection is Normal or an Attack. A Random Forest Classifier is trained on labeled historical traffic (KDD Cup '99 / UNSW-NB15) and deployed to classify live packets captured via Scapy.

02

Why This Problem Is Hard

⚡
Scale & Speed
Enterprise networks process millions of packets per second. Any detection system must be fast enough to classify in near real-time without creating a bottleneck.
⚖
Class Imbalance
In real traffic, attacks are rare compared to normal connections. A naive classifier that labels everything as "normal" achieves high accuracy but catches zero attacks.
🔀
Feature Heterogeneity
Network data mixes categorical features (protocol type, service) with continuous values (byte counts, rates). Preprocessing and encoding are non-trivial at scale.
🧬
Attack Diversity
Attacks span four major categories — DoS, Probe, R2L, U2R — each with dozens of sub-types and distinct traffic signatures. A single model must generalise across all.
🎭
Evasion Techniques
Sophisticated attackers deliberately craft traffic to resemble legitimate connections, fragment payloads, or randomise timing to evade detection systems.
⚙
False Alarm Cost
Every false positive (normal traffic flagged as attack) triggers an alert and wastes analyst time. In a busy SOC, alert fatigue leads to real attacks being ignored.

Attack Taxonomy

Code Category Description Examples Goal
DoS Denial of Service Flood a target server with so many requests that it becomes unavailable to legitimate users. SYN flood, Ping of Death, smurf, teardrop, neptune Disrupt availability
Probe Reconnaissance Scan a network or host to identify open ports, services, and vulnerabilities before launching a targeted attack. nmap, ipsweep, portsweep, satan, mscan Map attack surface
R2L Remote to Local An attacker without a local account exploits a vulnerability to gain local user access to a target machine. FTP write, guess_passwd, imap, phf, spy, warezclient Gain local access
U2R User to Root A local user exploits a vulnerability to escalate privileges to superuser/root, gaining full system control. buffer_overflow, loadmodule, perl, rootkit, sqlattack Full system takeover
04

The ML Solution

✓
Frame as binary classification. Collapse 22 attack sub-types into a single "Attack" class (label = 1) versus "Normal" (label = 0). This makes the problem tractable while covering all major threat categories.
✓
Use a Random Forest classifier. Random Forests are highly accurate on tabular data, naturally handle mixed numeric/categorical features, provide feature importances for interpretability, and require minimal hyperparameter tuning.
✓
Address class imbalance. Use class_weight="balanced" so the model penalises missing a rare attack more than misclassifying a common normal connection.
✓
Evaluate with the right metrics. Accuracy alone is misleading on imbalanced data. The confusion matrix, ROC-AUC, precision, recall, and F1 score together give a complete picture of detection quality.
✓
Deploy for live detection via Scapy. The trained model is serialised with joblib and loaded by a Scapy-based sniffer that extracts KDD-style features from each captured packet and classifies it in real time.

Project Pipeline

Data Loading
Load KDD Cup '99 via sklearn built-in or UNSW-NB15 from Kaggle CSV. Decode byte strings, subsample for speed.
fetch_kddcup99()
STEP 02
Preprocessing
LabelEncode categorical columns. Create binary label. StandardScale all 41 features. Stratified train/test split 80/20.
LabelEncoder
STEP 03
Training
Fit RandomForestClassifier (100 trees, depth 20, balanced weights). Parallelised across all CPU cores via n_jobs=-1.
RandomForest
STEP 04

Confusion matrix, ROC-AUC, classification report. Feature importance plot. Score distribution histogram.
